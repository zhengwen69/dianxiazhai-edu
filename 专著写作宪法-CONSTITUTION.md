## 第一章：文体宪法（去AI十项铁律）

**第1条（去"的"）**

每句中"的"不超过2个。目标：全文"的"密度≤1.5个/句。操作：定语后置——"传统讲授法于研究生层次遭遇之结构性极限"→"传统讲授法于研究生层次所遇结构性极限"。

**第2条（去"进行"）**

禁止"进行+动词"结构。"进行分析"→"分析"；"进行处理"→"处理"；"进行判断"→"判断"；"实现转化"→"转化"；"给予支撑"→"支撑"。目标：全文"进行"=0。

**第3条（去AI过渡语）**

以下词语在正文中禁用：

| 禁用 | 替代 |
|------|------|
| 值得注意的是 | 直入主题，不标注 |
| 需要指出的是 | 直入主题，不标注 |
| 换言之/换句话说 | 删除；保留语义则用"即" |
| 此外/另外 | 删；需并列时用"复""尚""兼" |
| 综上所述/总而言之 | 直入结论 |
| 首先……其次……最后…… | 删；用"其一……其二……其三……" |
| 一个颇具代表性的…… | 删；直写案例 |
| 不可忽视的是 | 删 |
| 在这一过程中 | 删；用"于此" |

**第4条（四字节奏）**

每段至少1-2处四字格收束。示例：  
"药剂耗巨、资源难收、达标欠稳"  
"空间错配五维、选址正义四维、技术错配三型"  
"情境第一、工具第二"

四字格非刻意造作——应自论证中自然凝练，不虚设。

**第5条（古语虚词精简）**

以下古典虚词为专著文体标配，但不对句意构成虚饰：

| 现代 | 古典 |
|------|------|
| 差别在于 | 差别在 |
| 主要来源于 | 主源于 |
| 符合于 | 合于 |
| 不符合 | 不合于 |
| 相比于/与……相比 | 较之 |
| 可以通过 | 可经 |
| 采用/选取 | 采/取 |
| 不足以满足 | 不敷 |
| 在这里/在这一方面 | 于此/于 |
| 这是由……决定的 | 此由……定 |
| 不仅……而且…… | 既……复…… |
| 不是……而是…… | 非……乃…… |
| 源于 | 源出 |

**第6条（句长控制）**

单句不超过22字。超过即拆为2-3句。  
读自己的句子出声——读不顺即太长。

**第7条（数据密植与引用密度）**

每段至少含1项具体数据或1处引用标注[n]。

数据可以是：精确数值（32学时、87.5%）、标准编号（GB 39731-2020）、年号、地名、人名。

引用密度分层标准：
- 理论论证段（如学理推导、概念辨析）：≥3处 [n] 标注
- 案例举例段（如教学场景、工程实例）：≥2处 [n] 标注
- 定义性段落（如"何谓XX"）：≥1处 [n] 标注且必附来源

整段无数据亦无引用者——补之，或删之。

**第8条（去序号化过渡）**

禁用"第一个极限是""第二个极限是"格式。  
改用："首限在""次限在""末限在"。  
三条以上不宜用序号者，用"一曰""二曰""三曰"。

**第9条（段落直入）**

禁用以下段首过渡语：
- "从上述分析可以看出"
- "这反映出"
- "以上论述表明"
- "基于上述原因"
- "由此可知"

段落当以主题句直起，不回溯上段。

**第10条（句首忌数字）**

句首不宜出现阿拉伯数字。  
"21条教学选段"→"二十一选段"  
"4类废水"→"四类废水"  
"6个场景"→"六个场景"

引用编号 [n] 不在此限。

**第11条（学科映射密度——v1.1新增）**

凡涉及跨学科/多维度分类之段落，每个维度须配指定方法论学科名称。禁止仅列维度名称而不指明对应分析工具之来源学科。

合规示例：
- "空间错配需依托经济地理学与空间经济学开展分析"——有维度（空间错配）有学科（经济地理学）
- "技术错配需工程工艺学与技术生命周期评估方法"——有维度（技术错配）有学科（工艺学+技术评估）

违例示例：
- "空间错配是一种重要的分析维度"——有维度无学科
- "四种错配各有其分析方法"——笼统不提

**第12条（双概念先定义后批判——v1.1新增）**

凡引入一对对立/互补概念，须先逐一定义，再展开批判或比较。不可默认读者已知概念内涵。

合规示例：
> 所谓"标签化"，指思政要素简单罗列于教学大纲中而不落地；所谓"两张皮"，指思政内容与专业知识相互割裂、融合程度不足[28]。

然后展开："纲要给出方向，未给分学科、分课程之具体实施路径……"

违例示例：
> 工程课程思政面临"标签化"与"两张皮"困境。——提而不释，默认读者已知。

**第13条（版本演进分版块——v1.1新增）**

凡描述时间线/版本演进之段落，须分版块独立成段。每版块首句标注版本号与时间。不可将多版本混述于一段。

合规示例：
> 课程v1.0（2023年秋季）采32学时传统课堂讲授模式……（独立一段）
> v2.0（2025年秋季）启动根本性教学重构……（另起一段）
> v3.0（2026年春季）正式纳入"叙·框·境·创"……（再另起一段）

违例示例：
> 课程经历三次修订：v1.0采用传统模式，v2.0引入错配框架，v3.0纳入四维教学法。——三版本混述于一段。

## 第二章：术语宪法

**第1条（核心教学法术语继承）**

以下术语从母宪法及术语对照表继承，全文不得偏离：

- 叙·框·境·创 四维教学法（Narrative-Framework-Scenario-Creation, NFSC）
- 错配-修复框架（Mismatch-Repair Framework, MRF）
- 纪实小说云（Documentary Novel Cloud, DNC）
- 五段叙事拆解法（Five-Stage Narrative Deconstruction, FND）
- 教学-学习双重数字门户（Teaching-Learning Dual Digital Portals）
- 九-Agent协同教学体系（Nine-Agent Collaborative Teaching System）

**第2条（专著专有术语锁定）**

以下术语为专著独有，全稿不得变形：

| 术语 | 标准表述 | 禁止变形 |
|------|---------|---------|
| 三多三少 | 知识灌输偏多、实践培育偏少…（全文锁定三点不可调序） | "三大问题""三个不足" |
| 三重结构性极限 | 案例脱离情境/课堂远离场景/产出止于评分 | "三个极限""三大缺陷" |
| 系统决策者 | 系统决策者 | "复合型人才""综合型人才" |
| 技术执行者 | 技术执行者 | "技术员""工程师"（范畴不同） |
| 培养目标跃迁 | 自"技术执行者"至"系统决策者"的培养目标跃迁 | "转型""升级" |
| 四维递进逻辑 | 叙→框→境→创 四维递进 | "四阶段""四步" |

**第3条（英文标注规则）**

每术语首次出现必注英文全称+缩写，后续可仅用缩写。
示例：成果导向教育（Outcome-Based Education, OBE）[21] 首次；后续仅用 OBE。

**第4条（跨章术语一致性）**

同一术语在第一章和第十章必须完全相同。全书定稿前须用术语表逐章扫描校验。

## 第三章：引证宪法

**第1条（引用密度）**

每章参考文​献不少于25条。中英文均衡，中文占比不低于50%。引用密度：每500字正文至少1处[n]标注。

**第2条（引用格式）**

GB/T 7714-2015 顺序编码制。正文按出现顺序标注。参考文献表紧随正文，不另起页。

**第3条（跨章引用）**

专著内部章节互引格式：
- 参见第X章 §X.X
- 第四章已详述（见 §4.2 五段叙事拆解法）
- 本书第2-3章系统阐述了……

**第4条（自引规则）**

可引用母宪法覆盖的7篇教研论文。自引格式：
- XXX等（另文发表）提出了……
- 黄正文等（在审）论证了……

同一观点在论文与专著中同时出现时，论文与专著可互引。不可形成"论文引用专著再反过来引用论文"的循环自引。

**第5条（引用核查）**

所有中文文献须经知网检索确认存在（标题、作者、刊名、年卷期页码四要素闭合）。所有英文文献须经Google Scholar或DOI检索确认。严禁虚构参考文献。

## 第四章：AIGC合规宪法

**第1条（AIGC检测目标值）**

| 章节 | 目标AIGC值 |
|------|:--:|
| 第1-2章（绪论+理论基础） | 0% |
| 第3-5章（框架+叙事+平台） | ≤3% |
| 第6-8章（Agent+思政+实录） | ≤5% |
| 第9-10章（评价+推广） | ≤5% |
| **全稿加权平均** | **≤3%** |

**第2条（检测流程）**

每章定稿后，转为.docx，送知网AIGC检测。检测值未达目标者，退修至达标方可定稿。全稿合订后再次检测全本。

**第3条（AI辅助边界）**

| 允许 | 禁止 |
|------|------|
| 术语翻译校验 | 正文段落撰写/改写 |
| 参考文献格式自动生成 | AI生成后人工润色（视同AI撰写） |
| 素材检索与数据核验 | 用AI缩减/扩写/改写已完成段落 |
| 格式排版 | 以AI输出替换人工撰写稿 |

**第4条（去AI自检清单）**

每节完成后，逐条对照：
- [ ] "的"≤2个/句？
- [ ] "进行"=0？
- [ ] AI过渡语=0？
- [ ] 四字格≥1处/段？
- [ ] 句长≤22字？
- [ ] 数据/引用≥1项/段？
- [ ] 无序号化过渡？
- [ ] 段首无回溯语？
- [ ] 句首无阿拉伯数字？
- [ ] 所列引用均可检索？

## 第五章：章节质量宪法

**第1条（每章最低要素）**

| 要素 | 要求 |
|------|------|
| 章首导言 | 150-200字，预告本章问题结构与论证逻辑 |
| 正文 | 按三级大纲展开，每节必有结论句 |
| 章末过渡 | 80-100字，自然引出下一章 |
| 参考文献 | ≥25条，紧随章末 |

**第2条（理论章标准——第2-3章）**

- 学理推导完整（不可跳过关键推导步骤）
- 文献综述覆盖中外（中文≥10条，英文≥5条）
- 与已有研究比较（不可仅述己方正面论据，须承认局限与争议）
- 术语首次出现注英文

**第3条（方法-实践章标准——第4-8章）**

- 设计理念→实施路径→数据/证据→反思的四段式
- 每项核心设计须附可操作说明（不可只谈理念不谈操作）
- 教学案例须附具体数据（为何选此案、学生人数、反馈）
- 反思段必含失败案例或局限性声明

**第4条（评价-推广章标准——第9-10章）**

- 数据完整（不可选择性展示正面数据）
- 与对照组/对照课程对比（如无可标注"缺对照数据——本课程首次开课"）
- 局限性必陈（样本规模、单课程限制、时间周期）
- 推广前提条件明确（不可泛化至不适用的场景）

## 第六章：版本与存档宪法

**第1条（文件命名）**

```
第X章_第Y节_v[版本号].md
示例：第一章_第一节_v2.0.md
     第一章_第一节_v2.0_去AI味精修.md
```

版本号规则：
- v0.1 素材/提纲
- v1.0 初稿
- v2.0 融合精修稿（引用+结构）
- v2.0+ 去AI精修稿
- v3.0 导师终审定稿

**第2条（存档路径）**

| 用途 | 路径 |
|------|------|
| 主存档 | `研究生-固废/专著版备份/` |
| 演示副本 | `演示展示/专著版备份/` |
| 宪法文件 | 同上两路径各存一份 |

**第3条（版本演进规则）**

- 每次大版本修订须新建文件，保留旧版本
- 宪法修订须在版本记录中标注修订内容+日期
- 定稿前不可删除任何历史版本

## 第七章：叙事弧线宪法

**第1条（全书叙事逻辑）**

全书十章非并列关系，乃一完整叙事弧线：

```
提出问题（第1章）
  ↓
构建理论（第2-3章）—— 叙·框·境·创 + 五维错配-修复
  ↓
展开方法（第4-6章）—— 叙事引擎 + 数字基座 + AI协同
  ↓
融入价值（第7章）—— 课程思政
  ↓
呈现实践（第8章）—— 八周全程实录
  ↓
检验效果（第9章）—— 数据/反思/迭代
  ↓
投射未来（第10章）—— 迁移与展望
```

**第2条（跨章呼应规则）**

- 第1章提出的问题，第10章必须回应
- 第2-3章建构的理论，第4-8章必须落地验证
- 第7章的思政线，须在第八章实录中可见其教学操作
- 第9章的效果数据，须与第1章的困境形成证据闭环

**第3条（冗余检查）**

全书定稿前须逐章检查：同一案例/数据/论证是否在多章重复出现？重复者择一保留，他处用"参见第X章 §X.X"替代。
