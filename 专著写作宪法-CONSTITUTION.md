## 第一章：文体宪法（去AI十项铁律）

**第1条（去"的"）**

每句中"的"不超过2个。目标：全文"的"密度≤1.5个/句。操作：定语后置——"传统讲授法于研究生层次遭遇之结构性极限"→"传统讲授法于研究生层次所遇结构性极限"。

**第2条（去"进行"）**

禁止"进行+动词"结构。"进行分析"→"分析"；"进行处理"→"处理"；"进行判断"→"判断"；"实现转化"→"转化"；"给予支撑"→"支撑"。目标：全文"进行"=0。

**第3条（去AI过渡语）**

以下词语在正文中禁用：

| 禁用 | 替代 |
|------|------|
| 值得注意的是 | 直入主题，不标注 |
| 需要指出的是 | 直入主题，不标注 |
| 换言之/换句话说 | 删除；保留语义则用"即" |
| 此外/另外 | 删；需并列时用"复""尚""兼" |
| 综上所述/总而言之 | 直入结论 |
| 首先……其次……最后…… | 删；用"其一……其二……其三……" |
| 一个颇具代表性的…… | 删；直写案例 |
| 不可忽视的是 | 删 |
| 在这一过程中 | 删；用"于此" |

**第4条（四字节奏）**

每段至少1-2处四字格收束。示例：  
"药剂耗巨、资源难收、达标欠稳"  
"空间错配五维、选址正义四维、技术错配三型"  
"情境第一、工具第二"

四字格非刻意造作——应自论证中自然凝练，不虚设。

**第5条（古语虚词精简）**

以下古典虚词为专著文体标配，但不对句意构成虚饰：

| 现代 | 古典 |
|------|------|
| 差别在于 | 差别在 |
| 主要来源于 | 主源于 |
| 符合于 | 合于 |
| 不符合 | 不合于 |
| 相比于/与……相比 | 较之 |
| 可以通过 | 可经 |
| 采用/选取 | 采/取 |
| 不足以满足 | 不敷 |
| 在这里/在这一方面 | 于此/于 |
| 这是由……决定的 | 此由……定 |
| 不仅……而且…… | 既……复…… |
| 不是……而是…… | 非……乃…… |
| 源于 | 源出 |

**第6条（句长控制）**

单句不超过22字。超过即拆为2-3句。  
读自己的句子出声——读不顺即太长。

**第7条（数据密植与引用密度）**

每段至少含1项具体数据或1处引用标注[n]。

数据可以是：精确数值（32学时、87.5%）、标准编号（GB 39731-2020）、年号、地名、人名。

引用密度分层标准：
- 理论论证段（如学理推导、概念辨析）：≥3处 [n] 标注
- 案例举例段（如教学场景、工程实例）：≥2处 [n] 标注
- 定义性段落（如"何谓XX"）：≥1处 [n] 标注且必附来源

整段无数据亦无引用者——补之，或删之。

**第8条（去序号化过渡）**

禁用"第一个极限是""第二个极限是"格式。  
改用："首限在""次限在""末限在"。  
三条以上不宜用序号者，用"一曰""二曰""三曰"。

**第9条（段落直入）**

禁用以下段首过渡语：
- "从上述分析可以看出"
- "这反映出"
- "以上论述表明"
- "基于上述原因"
- "由此可知"

段落当以主题句直起，不回溯上段。

**第10条（句首忌数字）**

句首不宜出现阿拉伯数字。  
"21条教学选段"→"二十一选段"  
"4类废水"→"四类废水"  
"6个场景"→"六个场景"

引用编号 [n] 不在此限。

**第11条（引号编码铁律——v1.6新增）**

此条为本次最纠结之排版纠错的宪法固化。

① **源文件允许**：`.md` 源文件中行文可暂用半角直引号 `"`（U+0022），降低写作输入负担。

② **排版脚本须转**：`.docx` 生成时，`publish` 脚本必须内置 `fix_chinese_quotes()` 函数：检测段落含中文字符者，将所有 `"` 按奇偶位置替换为 `\u201c`（左）和 `\u201d`（右）。英文段落及参考文献区保留原 `"` 不动。

③ **标题必须通转**：章标题、节标题、子节标题、四级标题——所有层级标题文本均须经 `fix_chinese_quotes()` 处理。不可遗漏任何一级。

④ **句末标点必剥**：四级标题末尾之 `。`（U+3002）须在输出前 `rstrip('\u3002')`。标题非正文，不随句号。

⑤ **禁用 `\u` 转义符（v1.8强化）**：编辑工具中的 `\u201c` 语法**不会**转为 Unicode 字符，将留下字面 `\u201c` 文本，被排版机检 A11 判定为异常字符。禁止在任何工具中以 `\uXXXX` 形式试图手动修复引号——此为本日实践反复确认之铁律。唯一修复路径：`publish` 脚本内置 `fix_chinese_quotes()` 函数自动转换。

⑥ **A11 检测**：排版机检 A11 量化验证——正文区 `"` 与 `"` 必须配对且 >0，straight `"` ≤ 10 且仅限英文语境。

**第12条（学科映射密度——v1.1新增）**

凡涉及跨学科/多维度分类之段落，每个维度须配指定方法论学科名称。禁止仅列维度名称而不指明对应分析工具之来源学科。

合规示例：
- "空间错配需依托经济地理学与空间经济学开展分析"——有维度（空间错配）有学科（经济地理学）
- "技术错配需工程工艺学与技术生命周期评估方法"——有维度（技术错配）有学科（工艺学+技术评估）

违例示例：
- "空间错配是一种重要的分析维度"——有维度无学科
- "四种错配各有其分析方法"——笼统不提

**第13条（双概念先定义后批判——v1.1新增）**

凡引入一对对立/互补概念，须先逐一定义，再展开批判或比较。不可默认读者已知概念内涵。

合规示例：
> 所谓"标签化"，指思政要素简单罗列于教学大纲中而不落地；所谓"两张皮"，指思政内容与专业知识相互割裂、融合程度不足[28]。

然后展开："纲要给出方向，未给分学科、分课程之具体实施路径……"

违例示例：
> 工程课程思政面临"标签化"与"两张皮"困境。——提而不释，默认读者已知。

**第14条（版本演进分版块——v1.1新增）**

凡描述时间线/版本演进之段落，须分版块独立成段。每版块首句标注版本号与时间。不可将多版本混述于一段。

合规示例：
> 课程v1.0（2023年秋季）采32学时传统课堂讲授模式……（独立一段）
> v2.0（2025年秋季）启动根本性教学重构……（另起一段）
> v3.0（2026年春季）正式纳入"叙·框·境·创"……（再另起一段）

违例示例：
> 课程经历三次修订：v1.0采用传统模式，v2.0引入错配框架，v3.0纳入四维教学法。——三版本混述于一段。

**第15条（段落类型分层标准——v1.2新增）**

全书段落分为三种类型，各类引用密度与文体优先规则不同：

| 类型 | 适用场景 | 引用密度 | 文体优先 |
|------|---------|:--:|------|
| **论证型** | 学理推导、概念辨析、比较分析 | ≥3处/段 | 引用=节奏 |
| **叙述型** | 版本演进、教学实录、案例描述 | ≥2处/段 | 引用≈节奏 |
| **指引型** | 导读、建议、操作指南、逻辑说明 | ≥1处/段 | **节奏>引用** |

指引型段落之核心学术权威来自结构清晰与措辞凝练——四字对仗、排比句、递进句等文体精华为首要追求。引用为锦上添花之辅助，不可为凑引用密度而破坏文体节奏。

合规示例（指引型）：
> 该门户可视为本书之动态活体附录——书中所论，门中可见；门中所示，书中有释。

违例示例（指引型过度引证）：
> 该门户可视为本书之动态活体附录[20]——书中论述之理论、架构与流程均可在门户中直观呈现[54]，门户内之功能模块与展示内容亦均在书中作出对应解读[58]。

## 第三章：引证宪法

**第1条（引用密度）**

每章参考文​献不少于25条。中英文均衡，中文占比不低于50%。引用密度：每500字正文至少1处[n]标注。

**第2条（引用格式）**

GB/T 7714-2015 顺序编码制。正文按出现顺序标注。参考文献表紧随正文，不另起页。

**第3条（跨章引用）**

专著内部章节互引格式：
- 参见第X章 §X.X
- 第四章已详述（见 §4.2 五段叙事拆解法）
- 本书第2-3章系统阐述了……

**第4条（自引规则）**

可引用母宪法覆盖的7篇教研论文。自引格式：
- XXX等（另文发表）提出了……
- 黄正文等（在审）论证了……

同一观点在论文与专著中同时出现时，论文与专著可互引。不可形成"论文引用专著再反过来引用论文"的循环自引。

**第5条（引用核查）**

所有中文文献须经知网检索确认存在（标题、作者、刊名、年卷期页码四要素闭合）。所有英文文献须经Google Scholar或DOI检索确认。严禁虚构参考文献。

## 第五章：章节质量宪法

**第1条（每章最低要素）**

| 要素 | 要求 |
|------|------|
| 章首导言 | 150-200字，预告本章问题结构与论证逻辑 |
| 正文 | 按三级大纲展开，每节必有结论句 |
| 章末过渡 | 80-100字，自然引出下一章 |
| 参考文献 | ≥25条，紧随章末 |

**第2条（理论章标准——第2-3章）**

- 学理推导完整（不可跳过关键推导步骤）
- 文献综述覆盖中外（中文≥10条，英文≥5条）
- 与已有研究比较（不可仅述己方正面论据，须承认局限与争议）
- 术语首次出现注英文

**第3条（方法-实践章标准——第4-8章）**

- 设计理念→实施路径→数据/证据→反思的四段式
- 每项核心设计须附可操作说明（不可只谈理念不谈操作）
- 教学案例须附具体数据（为何选此案、学生人数、反馈）
- 反思段必含失败案例或局限性声明

**第4条（评价-推广章标准——第9-10章）**

- 数据完整（不可选择性展示正面数据）
- 与对照组/对照课程对比（如无可标注"缺对照数据——本课程首次开课"）
- 局限性必陈（样本规模、单课程限制、时间周期）
- 推广前提条件明确（不可泛化至不适用的场景）

## 第七章：叙事弧线宪法

**第1条（全书叙事逻辑）**

全书十章非并列关系，乃一完整叙事弧线：

```
提出问题（第1章）
  ↓
构建理论（第2-3章）—— 叙·框·境·创 + 五维错配-修复
  ↓
展开方法（第4-6章）—— 叙事引擎 + 数字基座 + AI协同
  ↓
融入价值（第7章）—— 课程思政
  ↓
呈现实践（第8章）—— 八周全程实录
  ↓
检验效果（第9章）—— 数据/反思/迭代
  ↓
投射未来（第10章）—— 迁移与展望
```

**第2条（跨章呼应规则）**

- 第1章提出的问题，第10章必须回应
- 第2-3章建构的理论，第4-8章必须落地验证
- 第7章的思政线，须在第八章实录中可见其教学操作
- 第9章的效果数据，须与第1章的困境形成证据闭环

**第3条（冗余检查）**

全书定稿前须逐章检查：同一案例/数据/论证是否在多章重复出现？重复者择一保留，他处用"参见第X章 §X.X"替代。

## 第八章：排版校审宪法

**第16条（自动机检——v1.3新增）**

每章 .docx 生成后，须经排版自动检测脚本（`排版校审脚本/check_formatting.py`）扫描。检测标准见附录C《化工社排版自动机检清单》。全部10项通过方可进入人工目检。任一项"错误"级须退回修正。

**第17条（人工目检——v1.3新增）**

自动机检通过后，须开启 .pdf 逐页翻阅，按附录D《人工目检清单》逐项标记。目检发现排版缺陷，须修正后重新生成 .docx+.pdf，并重新执行第15条机检。

**第18条（排版校审通过标志——v1.3新增）**

同时满足以下条件，方为该章排版校审通过：
① 自动机检10项全部"通过"；
② 人工目检5项全部合格；
③ 机检报告与目检标记存档于 `专著版备份/校审记录/`。

全稿合订前，须归纳全章机检报告，出具《全稿排版校审总报告》。

**第19条（参考文献核查——v1.4新增）**

每章参考文献须经三级核查：

① **自动化核查**（A+B+C级）：运行 `check_references.py`，检测序号连续性（A1）、首次出现序一致性（A2）、悬空引用（B1）、死文献（B2）、标点格式（C1）、作者格式（C2）、四要素完整性（C3）。全部通过方可进入D级。

② **半自动真实性核查**（D级）：中文文献运行 `verify_refs_cnki.py` 生成CNKI检索链接，人工逐条点击确认文献真实存在；英文文献运行 `verify_refs_doi.py` 通过CrossRef API验证DOI。核查结果逐条标记于 `校审记录/参考文献核查_第X章.md`。

③ **人工内容核查**（E级）：由审稿人/导师逐条确认被引文献是否真实支撑正文所引论断。此级不可自动化。

核查报告存档于 `校审记录/`，与排版机检报告并列，同属该章定稿前置条件。

**第20条（理论章文献双轨制——v1.7新增）**

凡理论溯源类章节（如第二章四维教学法理论溯源、第三章错配框架学理推导等）引用国际经典教育理论（如Bruner、Piaget、Lave & Wenger等），须同时配该理论在中国学术语境中已获认可之中文综述或应用文献。国际经典提供理论原点，中文文献提供本土学术合法性。二者缺一不可。

合规示例（第二章）：
- 引Bruner (1991)叙事理论→配陈琦刘儒德(2020)《当代教育心理学》中叙事学习综述
- 引von Glasersfeld (1989)激进建构主义→配陈琦张建伟(1998)建构主义学习观要义评析
- 引Lave & Wenger (1991)情境学习→配王文静(2005)情境学习理论研究综述

违例示例：
- 仅引英文经典，全文无一中文教育理论文献配套（v1.0状态）
- 中文文献仅列《教育学》《教学论》通用教材，未针对性配套

本章节（理论章）参考文献中，中文教育理论文献占比不低于40%。

**第21条（排版管道自愈原则——v1.8新增）**

正文排版错误（引号半角、标题格式、标点残留、表分隔符泄漏）**不得**在 `.md` 源文件中"逐处手工修复"，必须由 `publish` 脚本统一后处理。源文件允许合理的格式瑕疵——排版脚本为唯一的质量闸门。

| 错误类型 | 违规操作 | 合规操作 |
|------|------|------|
| 引号半角 | 逐处手动替换 | `fix_chinese_quotes()` 一行解决 |
| 标题句号残留 | 逐处手动删除 | `rstrip('\u3002')` 一行解决 |
| 表格 `|---|` 泄漏至 Word | 手动删除 | `re.match(...)` 过滤一行 |
| 章标题重复 | 逐次合并后手动删除 | `merge` 脚本模板 `re.sub` 剥离 |

违例：发现一处引号半角→手动打开 .md 修改→合并→再发现一处→再手动修改（今日耗费数小时）。
合规：`publish` 脚本启动即自动修复上述四类问题，末节机检验证——30秒闭环。

**第22条（合并脚本模板化——v1.8新增）**

每章的 `merge_chX.py` 必须遵循固定模板：

```python
# ① 章标题
all_body = "# 第X章　标题\n\n"

# ② 剥离节文件中的章标题（强制）
text = re.sub(r'^#\s+第X章[^\n]*\n+', '', text, count=1)

# ③ 参考文献去重
# ④ 输出合并 .md
```

违例：手工拼接、忘记剥离标题→章标题重复（今日反复出现5次）。

**第23条（参考文献中期策略——v1.8新增）**

写作阶段与定稿阶段采用不同策略：

| 阶段 | 策略 | 允许 |
|------|------|:--:|
| **写作**（Ch1-10 分章撰写） | 每章独立编号，允许空号。删除文献时不重编其他序号 | ✅ 空号 |
| **定稿**（10章齐全后） | 脚本一次性全局去重+按首次出现顺序压缩编号 | ✅ 连续 |

此策略避免写作中途删改文献需逐章逐段重编引用的"连锁编辑灾难"。
